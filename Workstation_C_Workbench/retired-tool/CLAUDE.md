## WHAT THIS IS
*(EXAMPLE — the Dead / own-reference case.)* A small tool that was built, finished, and **retired** — kept
only as a **static reference record** of what it was and why it's done. Not active, not maintained, earns no
synthesis: a Dead project is a frozen record, not a live workspace.

This is the **own-reference** sub-kind in its **terminal state**. Contrast its Workbench siblings:
`reference-library/` (own-reference, **Active** — a library you still consult) and `build-tool/`
(Build/Workflow, **Active** — a tool you're still building). Same family, opposite end of the lifecycle.

## STATE GATES STRUCTURE
A Dead project gets the minimum: this `CLAUDE.md` + a single record. **No `CONTEXT.md` synthesis** (nothing
live to synthesize), no `_reasoning-log/`, no `references/`. The doctrine's rule made concrete — structure is
earned by *Active* state; a Dead project sheds it.
